# CRM Кафе

Система управления банкетным обслуживанием для кафе/ресторана. Позволяет планировать мероприятия, рассчитывать потребность в ингредиентах, отслеживать запасы и формировать закупки.

## Возможности

- **Мероприятия** — создание и управление банкетами (клиент, дата, время, количество гостей, статус)
- **Меню** — составление меню из блюд с указанием порций на мероприятие
- **Расчёт ингредиентов** — автоматический подсчёт необходимых продуктов на основе меню и количества гостей
- **Запасы** — учёт продуктов в холодильниках с контролем сроков годности
- **Список закупок** — формирование списка покупок с учётом имеющихся остатков
- **Закупки** — учёт закупленных продуктов, автосписание заморозки в запасы
- **Календарь** — наглядное отображение мероприятий по датам (FullCalendar)
- **Дашборд** — сводка по активным мероприятиям и последним закупкам

## Статусы мероприятий

- `new` — новое
- `confirmed` — подтверждено
- `in_progress` — в процессе
- `completed` — завершено
- `cancelled` — отменено

## Требования

- PHP >= 8.3
- Node.js >= 20
- Composer

## Установка

```bash
composer setup
```

Команда выполнит:
1. Установку PHP-зависимостей
2. Создание `.env` из `.env.example`
3. Генерацию ключа приложения
4. Миграцию базы данных (SQLite)
5. Установку npm-зависимостей
6. Сборку фронтенда (Vite)

## Загрузка демо-данных

```bash
php artisan db:seed --class=DemoDataSeeder
```

## Запуск

```bash
composer dev
```

Запускает одновременно:
- **PHP-сервер** (http://localhost:8000)
- **Очередь** (queue:listen)
- **Логи** (pail)
- **Vite Dev Server** для горячей перезагрузки фронтенда

## Структура

```
app/
├── Http/Controllers/    — контроллеры CRUD
├── Models/              — Eloquent-модели
└── Services/            — бизнес-логика (расчёт ингредиентов)

resources/views/
├── events/              — мероприятия (включая календарь)
├── dishes/              — блюда
├── ingredients/         — ингредиенты
├── refrigerators/       — холодильники
├── inventory/           — запасы
└── purchases/           — закупки

routes/web.php           — все маршруты
```

## База данных

По умолчанию используется **SQLite** (`database/database.sqlite`).

Основные таблицы:
- `events` — мероприятия
- `dishes` — блюда
- `ingredients` — ингредиенты
- `event_dish` — связь мероприятие→блюдо (с количеством порций)
- `dish_ingredient` — связь блюдо→ингредиент (с количеством на порцию)
- `refrigerators` — холодильники
- `inventories` — остатки продуктов
- `purchases` — закупки
- `purchase_items` — позиции закупки
