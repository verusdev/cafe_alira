# CRM Кафе

Система управления банкетным обслуживанием для кафе/ресторана. Позволяет планировать мероприятия, рассчитывать потребность в ингредиентах, отслеживать запасы, формировать закупки и вести финансовый учёт.

## Роли

| Роль | Описание | Права |
|------|----------|-------|
| **order_taker** | Принимающий заявки | Мероприятия (CRUD), создание заявок с лендинга |
| **cook** | Повар | Блюда, ингредиенты, холодильники, запасы, закупки |
| **manager** | Руководитель | Всё + финансы, Excel-отчёты |

## Возможности

### Мероприятия
- Создание, редактирование, просмотр, удаление
- Типы: банкет, свадьба, корпоратив, фуршет, кофе-брейк, другое
- Статусы: новое → подтверждено → в процессе → завершено / отменено
- Валидация переходов статусов (нельзя вернуть из completed/cancelled)
- **Календарь** (FullCalendar, день/неделя/месяц, цвета по статусу)
- **Поиск и фильтры** — по имени/телефону, статусу, типу, диапазону дат

### Меню и ингредиенты
- Каталог блюд с категориями и ценой за порцию
- Каталог ингредиентов с единицами измерения
- Привязка ингредиентов к блюдам (количество на порцию)
- Автоматический расчёт требуемых ингредиентов на мероприятие

### Склад и закупки
- Холодильники с учётом остатков
- Список закупок (что нужно купить с учётом остатков)
- Создание и завершение закупок
- Статусы закупок: новый → в процессе → завершён

### Финансы (только руководитель)
- Стоимость услуги (базовая цена типа × гости)
- Стоимость меню (блюда × порции)
- Затраты на продукты
- Ожидаемая прибыль
- Финансовая сводка на дашборде
- Рентабельность

### Повторяющиеся клиенты
- Автоматическое определение при создании/редактировании мероприятия
- Предупреждение в форме: «Клиент уже обращался ранее»
- Блок «Клиент также заказывал» на странице мероприятия
- 🔄 кнопка в списке для быстрого создания повторного заказа

### Печать
- **Меню для клиента** — чистая вёрстка, скрытие элементов управления при печати
- **Чек-лист закупок** — с чекбоксами, отметка купленного

### Публичный лендинг
- `/` — hero, услуги, цены, форма заявки
- Карточки цен при клике выбирают тип в форме
- Создаёт мероприятие со статусом `new`
- Уведомляет order_taker и manager

### REST API (Sanctum)
- `POST /api/login` — получение токена
- `POST /api/logout` — выход
- `GET /api/user` — текущий пользователь
- `GET /api/events` — список мероприятий (с пагинацией)
- `POST /api/events` — создание
- `GET /api/events/{id}` — детально + финансы
- `PUT /api/events/{id}` — обновление
- `DELETE /api/events/{id}` — удаление
- `GET /api/events/{id}/shopping-list` — список закупок

### Экспорт (только руководитель)
- `GET /export/events` — Excel-файл со всеми мероприятиями
- `GET /export/finance` — Excel-файл с финансовым отчётом

### Уведомления
- 🔔 колокольчик в шапке счётчиком непрочитанных
- Страница `/notifications` со списком
- Прочитать одно / все
- Уведомление о новой заявке с лендинга

## Статусы мероприятий

| Код | Метка | Цвет |
|-----|-------|------|
| `new` | Новый | Синий |
| `confirmed` | Подтверждён | Зелёный |
| `in_progress` | В процессе | Жёлтый |
| `completed` | Завершён | Серый |
| `cancelled` | Отменён | Красный |

## Статусы закупок

| Код | Метка |
|-----|-------|
| `pending` | Новый |
| `in_progress` | В процессе |
| `completed` | Завершён |

## Технологии

- **Laravel 13** — PHP-фреймворк
- **Tailwind CSS 4** — стилизация
- **FullCalendar 6** — календарь
- **Vite 8** — сборка фронтенда
- **SQLite** — база данных
- **PhpSpreadsheet 5** — Excel-отчёты
- **Laravel Sanctum** — API-аутентификация
- **PHPUnit** — тесты (42 теста, 86 ассертов)

## Требования

- PHP >= 8.3
- Node.js >= 20
- Composer

## Установка

```bash
composer setup
```

Команда выполнит:
1. Установку PHP-зависимостей
2. Создание `.env` из `.env.example`
3. Генерацию ключа приложения
4. Миграцию базы данных (SQLite)
5. Установку npm-зависимостей
6. Сборку фронтенда (Vite)

## Запуск через Docker

```bash
docker compose up -d
```

Приложение будет доступно на http://localhost.

Команда выполнит:
1. Сборку PHP 8.3 FPM образа с расширениями (sqlite, gd, mbstring и др.)
2. Установку Composer-зависимостей
3. Генерацию ключа приложения
4. Создание и миграцию SQLite базы данных
5. Установку npm-зависимостей и сборку Vite
6. Запуск Nginx на порту 80

Остановка:
```bash
docker compose down
```

## Запуск без Docker

```bash
composer dev
```

Запускает одновременно:
- **PHP-сервер** (http://localhost:8000)
- **Очередь** (queue:listen)
- **Логи** (pail)
- **Vite Dev Server** для горячей перезагрузки фронтенда

## База данных

По умолчанию используется **SQLite** (`database/database.sqlite`).

Основные таблицы:
- `events` — мероприятия
- `dishes` — блюда
- `ingredients` — ингредиенты
- `event_dish` — связь мероприятие→блюдо (с количеством порций)
- `dish_ingredient` — связь блюдо→ингредиент (с количеством на порцию)
- `refrigerators` — холодильники
- `inventories` — остатки продуктов
- `purchases` — закупки
- `purchase_items` — позиции закупки
- `notifications` — уведомления
- `personal_access_tokens` — токены API (Sanctum)
